# 從零開始精通：Alpha-Beta 剪枝演算法 (Alpha-Beta Pruning)

這份教學專門為「完全沒有演算法背景」的開發者所撰寫。讀完這篇文章，你將能深刻理解 Alpha-Beta 剪枝的原理，並知道如何在棋盤遊戲（如井字遊戲、五子棋、象棋）中使用它來打造「絕對不會下錯步」的完美 AI。

---

## 🧭 第一部分：暴力破解法 —— 對抗搜尋 (Minimax)

在學「剪枝」之前，我們得先了解這棵樹是怎麼長出來的。

想像你（Max）跟一個極端聰明且惡劣的對手（Min）在玩一個闖關遊戲。
這個遊戲是回合制的：**第一回合你選路，第二回合對手選路，第三回合遊戲結束並結算分數。**

為了贏得最高分，你決定在腦中「推演未來所有的可能性」，畫出一棵**賽局樹 (Game Tree)**：
1. **你的回合 (Max Node)**：你想拿**最高**的分數。
2. **對手的回合 (Min Node)**：對手一定會想盡辦法把你的分數壓到**最低**。

### Minimax 的運作方式
你發現面前有「左」、「右」兩道門：
*   **推演左邊的門：** 你進去後，對手面臨兩個寶箱。一個裡面有 `$10`，一個有 `$100`。因為對手很惡劣 (Min)，所以他一定會給你 `$10`。
*   **推演右邊的門：** 你進去後，對手面臨兩個寶箱。一個裡面有 `$5`，一個有 `$20`。對手一定會給你 `$5`。

綜合以上的腦內推演，你知道：
*   選左邊 ➡️ 對手會給你 `$10`。
*   選右邊 ➡️ 對手會給你 `$5`。
因為你是想拿高分的 Max，所以你最後會決定：**走左邊的門！**這就是大名鼎鼎的 **Minimax 演算法**。

## ✂️ 第二部分：聰明破解法 —— Alpha-Beta 剪枝

Minimax 雖然能保證找出最佳解，但如果遊戲很複雜（例如西洋棋的未來分支有幾千億種），你的大腦（或電腦）根本算不完所有的可能性。這時候就需要 **Alpha-Beta 剪枝 (Alpha-Beta Pruning)**。

### 什麼是「剪枝 (Pruning)」？
剛才的故事我們換個情境來推演：
1. 你先去左邊房間推演，看完發現對手最狠也會給你 **`$10`**。（所以你現在的「底線」是 `$10`）
2. 接著你走到右邊房間推演，剛翻開**第一個寶箱發現只有 `$2`**。

🚨 **這時候演算法會直接停止搜尋右邊房間！** 🚨

為什麼？
因為你知道對手一定會選 `$2` **或更低的** 給你（對手是 Min）。
既然右邊房間最多只會讓你拿到 `$2`，而左邊房間確保你能拿到 `$10`，那右邊房間剩下的寶箱不管是 `$100` 還是 `$0`，對你都已毫無意義！
這時候程式會直接把右邊剩下的分支「**剪掉 (Cut-off)**」不看，省下大把的計算力！

### 專業術語：Alpha ($\alpha$) 與 Beta ($\beta$)

為了把剛才的「底線」寫進程式裡，我們需要兩個變數隨著樹往下傳遞：
*   **$\alpha$ (Alpha)**：對於 Max 玩家來說，目前找到的**最保底 (最好)**的結果。初始值為 $-\infty$。
*   **$\beta$ (Beta)**：對於 Min 玩家來說，目前找到的**對 Max 最不利 (最差)**的結果。初始值為 $+\infty$。

**剪枝發生的條件 (Alpha $\ge$ Beta)：**
當搜尋過程中發生 $\alpha \ge \beta$ 時，代表當前節點以下的剩餘路徑，已經不可能再改變原本的決策了（對手已經找到一個夠爛的步可以逼退你了），因此可以直接中斷該節點的搜尋。

## 🕹️ 第三部分：井字遊戲中的實作設計

我們即將在 `board-game-rl` 的專案中實作 `AlphaBetaAgent`。在像井字遊戲這樣的小型（狀態空間不大）的遊戲中，Alpha-Beta 可以瞬間算到底，產生一個**絕對完美的對手（不可能輸的打法）**。

### 1. 估值函數 (Evaluation Function)

當賽局樹走到底部（遊戲結束）時，我們需要給這個結局一個分數：

```python
# _evaluate() 的核心邏輯
if winner == self.player:    # AI 贏了
    return +10.0
elif winner == -self.player: # 對手贏了
    return -10.0
elif winner == 0:             # 平手
    return 0.0
```

只有三種固定分數：`+10`、`-10`、`0`。

### 2. 「折扣懲罰」：讓 AI 傾向快贏

但你在 Log 裡會看到 `9.4`、`9.2` 這樣的小數，原因是每次遞迴往上傳遞的時候，都套了這段懲罰：

```python
if score > 0:
    score -= 0.1   # 每往上傳遞一層，勝利分數減 0.1
elif score < 0:
    score += 0.1   # 每往上傳遞一層，失敗分數加 0.1
```

**贏得越快，最終得分越高。** 這讓 AI 不只是「找到一條贏的路」，而是「選最快拿下勝利的路」：

| Log 中看到的得分 | 意義 |
|-----------------|------|
| `9.9` | 再 1 步就能贏 |
| `9.8` | 再 2 步能贏 |
| `9.4` | 再 6 步能贏 |
| `0.0` | 雙方完美發揮，這條路最終是平手 |
| `-9.x` | 這條路最終 AI 會輸 |


### 2. 優雅的紀錄觀察 (Logging)
為了觀察 Alpha-Beta 是怎麼在人類看不到的地方「剪掉」無用的未來，我們會在程式碼中加入樹狀的 Log：

```text
========================================
🧠 第 O 方 (AI) 開始腦內推演 (Alpha-Beta)
┣━ 🤖 AI (MAX) 嘗試下在 (0, 0) ...
  ┣━ 👤 Player (MIN) 嘗試下在 (0, 1) ...
  ┃  ┗━ 預期最終得分: 9.4 ➔ (目前保底 α=-inf, 防線 β=9.4)
┃  ┗━ 預期最終得分: 0.0 ➔ (目前保底 α=0.0, 防線 β=inf)
┣━ 🤖 AI (MAX) 嘗試下在 (0, 1) ...
  ┣━ 👤 Player (MIN) 嘗試下在 (0, 0) ...
  ┃  ┗━ 預期最終得分: 0.0 ➔ (目前保底 α=0.0, 防線 β=0.0)
  ✂️ ┗━ 【剪枝發生】對策太差 (α≥β)，跳過剩餘 7 種可能性！
✨ 推演結束！決定走 (0, 0)，預期得分：0.0
========================================
```

## 🔢 第四部分：井字遊戲的狀態空間分析（親手驗證）

理論歸理論，我們可以用程式碼親手算出這些數字來驗證。執行 `scripts/count_states.py`：

### Minimax vs Alpha-Beta 節點數（實際測量）

| 演算法 | 走過的節點數 | 節省比例 |

### 從 Log 中觀察到的三個重要結論

執行腳本後，你可以從 Alpha-Beta 的推理 Log 觀察到：

1. **井字遊戲的完美解是「平手」**
   空盤面的每一個起手選擇，AI 最終預期得分都是 `0.0`。這代表當雙方都完美發揮時，必然平手——沒有人可以強迫贏。

2. **剪枝效果立刻就發生**
   從 Log 可以看到，AI 在嘗試第二步 `(0,1)` 時，對手 MIN 只看了第一種回應就直接觸發 `【剪枝發生】跳過剩餘 7 種可能性！` ——這就是 Alpha-Beta 的威力！

3. **啟發式排序能讓剪枝更有效率**
   目前我們的動作排序是固定的 `(0,0), (0,1), ... (2,2)`。如果能讓更好的棋步排在前面（例如先試中心格），剪枝力度可以進一步提升。

## 📚 延伸學習

掌握了 Alpha-Beta 之後，所有雙人對戰、沒有運氣成分（沒有骰子）的棋盤類遊戲（如五子棋、黑白棋、象棋）你都可以寫出一個很強的 AI 了！

*   **啟發式評估函數 (Heuristic Evaluation)**：若遊戲太大算不到底，可在中途用規則給盤面打分，搭配 Alpha-Beta 繼續剪枝。
*   **反覆加深搜尋 (Iterative Deepening)**：先算深度 1，再算深度 2，……這樣即使時間到了也能回傳目前最佳解。
*   **神經網路估值函數**：讓機器學習模型取代手寫規則來評估盤面，這就是 AlphaGo 的雛形思想。
